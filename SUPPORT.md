# Support Guidelines

Thank you for using ProCode EduPulse! We are here to help you succeed.

## How to Get Support

### 1. Check the Documentation
Before reaching out, please check the [Documentation Hub](https://soghayarmahmoud.github.io/procode-edu-pulse-lms/#/docs) and the [Style Guide](https://soghayarmahmoud.github.io/procode-edu-pulse-lms/#/styleguide). Most common questions about the platform architecture and UI components are answered there.

### 2. GitHub Discussions
For general questions, "how-to" guides, and brainstorming new features, please use the [GitHub Discussions](https://github.com/soghayarmahmoud/procode-edu-pulse-lms/discussions) tab. This help build a searchable knowledge base for the entire community.

### 3. Open an Issue
If you find a bug or have a specific feature request, please [open an issue](https://github.com/soghayarmahmoud/procode-edu-pulse-lms/issues/new/choose) using one of our templates.

### 4. Direct Contact
For sensitive matters or formal inquiries, you can reach out to the maintainer via:
- **LinkedIn**: [Mahmoud ElSoghayar](https://linkedin.com/in/elsoghayar)
- **Email**: [support@procode.edu](mailto:support@procode.edu)

## Before You Ask

- **Be Specific**: Include version numbers, browser info, and clear steps to reproduce the issue.
- **Search First**: Check if a similar issue or discussion already exists.
- **Provide Context**: What were you trying to achieve? What actually happened?

## Contribution as Support
The best way to support the project is by contributing! See our [CONTRIBUTING.md](CONTRIBUTING.md) to get started.
