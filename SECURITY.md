# Security Policy

## Supported Versions

We currently provide security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 2.x.x   | :white_check_mark: |
| 1.x.x   | :x:                |

## Reporting a Vulnerability

We take the security of ProCode EduPulse seriously. If you believe you have found a security vulnerability, please report it to us as soon as possible.

**Do not open a public GitHub issue for security reports.**

Instead, please send an email to [security@procode.edu](mailto:security@procode.edu) (or contact the maintainer directly via LinkedIn).

Please include the following in your report:
- A description of the vulnerability.
- Steps to reproduce the issue.
- Potential impact of the vulnerability.
- Any suggested fixes (optional).

### Our Process

Once a report is received:
1. We will acknowledge receipt of your report within 48 hours.
2. We will investigate the issue and determine the severity.
3. We will keep you updated on our progress toward a fix.
4. Once a fix is ready, we will coordinate a release and credit you (if desired).

Thank you for helping keep ProCode EduPulse secure!
